# 차량 1대 처리 체크리스트 (수출 채널)

> **대상**: 수출 채널(export) 차량 1대를 신규 등록부터 거래완료까지 진행할 때 사용하는 운영 가이드.
> **범위**: 헤이맨·카풀 채널 제외. 정산은 거래완료 후 별도 진행이라 마지막 부록으로 분리.
> **작성 기준**: 2026-05-19 시점 구현 완료 큐 (큐 20-D 자금이체 5상태 게이트 + 매입·판매 재무 확정 분리까지).

## 작업자 표기 규칙

각 체크박스 앞에 작업자를 한 글자로 표기합니다.

| 표기 | 작업자 | 주 메뉴 |
|---|---|---|
| **[영]** | 영업 | 차량 관리, 바이어, 컨사이니, 영업담당자 |
| **[재]** | 재무 (정산 담당) | 재무 처리, 정산, 채권관리 |
| **[통]** | 통관 / 선적 | 차량 관리(수출통관·선적·DHL 탭), 포워딩사 |
| **[관]** | 관리자 (admin / super) | 승인 큐, 기능 설정 |

> 한 사람이 여러 역할(role)을 겸할 수 있습니다. 이 표기는 "그 권한을 가진 사람이 처리해야 한다"는 뜻이고, 작업자 본인의 직책과는 다를 수 있습니다.

## 차량 진행상태 10단계

각 단계는 차량 편집 패널의 입력값에 따라 시스템이 자동 계산합니다. 직접 바꿀 수 없습니다.

```
매입중 → 매입완료 → 말소완료 → 판매중 → 판매완료
  → 수출통관중 → 수출통관완료 → 선적중 → 선적완료 → 거래완료
```

---

## A. 영업 단계 (차량 신규 등록 → 매입 → 말소 → 판매)

### A-0. 사전 준비 (해당 차량 처리 전에 마스터 데이터가 등록되어 있어야 함)

- [ ] **[영]** 사이드바 → **바이어** 에서 거래 바이어가 등록되어 있는지 확인. 없으면 신규 등록.
- [ ] **[영]** 사이드바 → **컨사이니** 에서 컨사이니(수하인)가 등록되어 있는지 확인. 없으면 신규 등록.
- [ ] **[통]** 사이드바 → **포워딩사** 에서 운송 포워딩사가 등록되어 있는지 확인. 없으면 신규 등록.
- [ ] **[영]** 사이드바 → **영업담당자** 에서 본인이 영업담당자로 등록되어 있는지 확인. 없으면 관리자에게 요청.

> **참고**: 같은 바이어에 미수 잔존 차량이 있는 경우의 잠금은 **판매 단계(A-4)** 에서 바이어 배정 시점에 평가됩니다. 차량 신규 등록 시점에는 평가되지 않습니다.

### A-1. 차량 신규 등록 — [기본정보] 탭

- [ ] **[영]** 사이드바 → **차량 관리** → 우측 상단 **[+ 신규 등록]** 클릭 → 차량 편집 슬라이드 패널 열림.
- [ ] **[영]** [기본정보] 탭에서 다음 **필수 항목** 입력:
  - 차량번호
  - 제조사·차종·연식
  - 배기량(cc)·중량(kg)
  - 주행거리(km)
  - 색상
- [ ] **[영]** NICE 등록정보 12종(차대번호·소유자명·주민등록번호 등)은 **수동 입력**. (NICE API 자동조회는 향후 도입 예정 — 현재는 수동만)
- [ ] **[영]** **소유자 주민등록번호** 입력 (말소·등록증·양도증명서 PDF에 사용). 시스템이 자동 암호화 저장하므로 평문 노출 안 됨.
- [ ] **[영]** NICE 제원정보 12종(모델·변속기·구동방식·연비 등) 수동 입력. 채워두면 추후 서류 발급 정확도 향상.
- [ ] **[영]** 좌측 하단 **[저장]** 클릭 → 차량 생성. 진행상태 자동 계산 시작.

### A-2. 매입 정보 입력 — [매입] 탭

- [ ] **[영]** 차량 편집 패널 → [매입] 탭 열기.
- [ ] **[영]** 다음 **필수 항목** 입력:
  - 매입일
  - 영업담당자 (본인 또는 담당자 선택)
  - 매입처
  - 매입가
- [ ] **[영]** 매입 수수료(매도비) 입력. 필요 시 비용 9종(말소비·등록비·견인비·운반비·해상운송비·보험료·이전등록비·기타비용1·기타비용2) 입력.
- [ ] **[영]** 매입 선금(계약금) 지급 시 **선금** 항목 입력.
- [ ] **[영]** 매입 잔금 결제 시마다 [매입 잔금] 섹션에서 **[+ 잔금 추가]** → 결제일·금액 입력 → 저장.
- [ ] **[영]** 매입 잔금 저장 직후 해당 행의 진행상태는 **"대기"** (호박색 뱃지)로 표시됨. 이 상태에서는 미지급액에 반영되지 **않음**. 재무 확정 필요 (B-1 단계).
- [ ] **[영]** 차량 진행상태가 **매입중** 인지 화면 상단에서 확인.

### A-3. 말소 처리 — [매입] 탭 + [서류] 탭

> **선행 조건**: 차량 진행상태가 **매입완료** 에 도달한 후 진행하는 것이 자연스럽습니다. 매입완료는 B-1 매입 잔금 재무 확정이 끝나야 도달합니다. 시스템상 매입중 상태에서도 말소 처리 자체는 가능하지만, 매입 잔금이 미확정 상태에서 말소를 진행하면 회계 흐름이 꼬일 수 있습니다.

- [ ] **[영]** [서류] 탭 → **자동차말소등록신청서 (별지 제17호)** 다운로드.
- [ ] **[영]** 차량 등록증을 시청·구청에 제출하여 말소 절차 진행. (시스템 외부 행정 절차)
- [ ] **[영]** 말소가 완료되면 차량 편집 패널 → [매입] 탭에서 **말소 체크박스** 체크.
- [ ] **[영]** 같은 [매입] 탭의 **말소 서류 업로드** 영역에 시청에서 받은 말소필증 스캔본 업로드.

> **게이트 — 말소 mismatch 확인 모달**: 말소 체크박스만 켜거나 서류만 올린 상태로 저장하면 ⚠ **단계 진입 누락 확인** 모달이 뜸. "체크박스와 문서 업로드가 한쪽만 되어있어 해당 단계가 진행되지 않습니다" 안내 → 둘 다 처리해야 진행상태가 **말소완료** 로 넘어감.

- [ ] **[영]** 저장 후 차량 진행상태가 **말소완료** 로 바뀌었는지 확인.

### A-4. 판매 정보 입력 — [판매] 탭

- [ ] **[영]** 차량 편집 패널 → [판매] 탭 열기.
- [ ] **[영]** 다음 **필수 항목** 입력:
  - 판매일
  - 통화 (수출은 USD/JPY/EUR/GBP/CNY 중 선택)
  - 환율
  - 바이어
  - 컨사이니
  - 판매가
- [ ] **[영]** 필요 시 세금/할인·수수료·운송비·자동로딩·기타비용 입력. (분모는 부대비용 포함이라 정확 입력 권장)

> **게이트 — 같은 바이어 미수 잔존 (큐 14 G2)**: 바이어를 선택해 저장하는 순간 시스템이 평가합니다. 동일 바이어로 미수 잔존 차량이 있으면 차량 편집 패널 상단에 "이 바이어는 미수 잔존 차량이 있습니다. 신규 거래는 관리자 승인이 필요합니다" 경고 + 잠금. **[신규 거래 승인 요청]** 버튼 → [관] 승인 큐 → 관리자 승인 후 (해당 차량번호에 한해) 저장 가능.

- [ ] **[영]** 입금 받는 대로 [판매 잔금] 섹션에서 **[+ 잔금 추가]** → 입금일·금액 입력 → 저장.
- [ ] **[영]** 판매 잔금 행이 **"대기"** 호박색 뱃지로 표시되는지 확인. 미입금액에 반영되지 **않음**. 재무 확정 필요 (B-2 단계).
- [ ] **[영]** 선수금 1·2 또는 적립금 사용이 있으면 해당 칸 입력.
- [ ] **[영]** [판매] 탭 상단 **미납 비율 게이지** 가 0%에 가까워지는지 확인.

> **게이트 — 50% 룰**: 미납 비율이 **50% 초과** 상태에서는 B/L 발행이 잠금 처리됨 (C-4 단계). 그 전에 입금을 받아 미수율을 50% 이하로 낮추거나, 50% 룰 예외 승인을 받아야 함.

- [ ] **[영]** 저장. 차량 진행상태가 **판매중** 또는 **판매완료** 로 변경되는지 확인.

## B. 재무 단계 (영업 입력 → 재무 확정)

> 재무 단계는 영업 단계와 **병행**으로 진행됩니다. 영업이 잔금을 입력한 직후 재무가 들어가 확정하는 흐름. 큐 20에서 도입된 분리 구조라 영업이 잔금을 저장해도 곧바로 미수/미지급에 반영되지 않습니다. 재무가 확정해야 회계 수치에 반영됩니다.

### B-1. 매입 잔금 확정

- [ ] **[재]** 사이드바 → **재무 처리** 클릭. 사이드바 메뉴 옆 호박색 배지에 대기 건수가 표시됨.
- [ ] **[재]** 상단 탭 **[매입 잔금]** 선택.
- [ ] **[재]** 대상 차량의 매입 잔금 행 확인. 영업이 입력한 결제일·금액·매입처 표시됨.
- [ ] **[재]** **매입처 계좌** 정보 입력 (큐 20-C 도입). 시스템이 자동 암호화 저장.
- [ ] **[재]** 실 이체 처리 후 **[확정]** 버튼 클릭.

> **게이트 — self-confirm 차단**: 영업과 재무가 동일 인물이면 본인이 입력한 잔금은 본인이 확정 불가. 다른 권한자가 확정해야 함.

- [ ] **[재]** 또는, 잘못된 입력이라면 **[거부]** 클릭 → 거부 사유 5자 이상 입력 (필수). 행이 **재무 거부** 상태로 표시되고 영업이 재입력 가능.
- [ ] **[재]** 확정 후 잔금 행이 **"확정"** (녹색 뱃지)으로 바뀌고 확정자명·확정일시 표시되는지 확인.

> **회계 무결성 잠금**: 확정 처리된 잔금 행은 더 이상 금액 수정/삭제 불가. 수정 시도 시 잠금 토스트 표시. 변경 필요 시 별도 취소 이체로 처리.

### B-2. 판매 잔금 확정

- [ ] **[재]** 사이드바 → **재무 처리** → 상단 탭 **[판매 잔금]** 선택.
- [ ] **[재]** 대상 차량의 판매 잔금 행 확인.
- [ ] **[재]** 입금 확인 후 **[확정]** 클릭 또는 잘못된 입력이면 **[거부]** + 사유 입력.
- [ ] **[재]** 확정 후 차량 진행상태가 **판매중 → 판매완료** 로 자동 전환되는지 확인 (미입금 ≤ 0 도달 시).
- [ ] **[재]** 채권관리 메뉴에서 해당 차량의 미수금이 감소했는지 교차 확인 (선택).

### B-3. (예외 흐름) 이미 확정된 잔금 되돌리기 — 취소 이체

- [ ] **[재/영]** 이미 확정된 잔금을 되돌려야 하는 경우, 차량 패널에서 **취소 요청** → [관]에게 승인 요청 → [관] 승인 → 재무 처리 화면에 **취소 ⊘ 대기** 표시 → 재무가 다시 **[확정]** 또는 **[거부]** 처리. 거부 시 **취소 거부 확인** 모달 표시.

## C. 통관 + 수출 + B/L 단계

### C-1. 수출통관 정보 입력 — [수출통관] 탭

- [ ] **[통]** 차량 편집 패널 → [수출통관] 탭 열기.
- [ ] **[통]** 다음 **필수 항목** 입력:
  - 수출 바이어
  - 수출 컨사이니
  - 포워딩사
  - 선적일
  - 도착예정일
  - 선적방법 (RORO 또는 CONTAINER)
- [ ] **[통]** **수출 바이어 + 선적일** 두 항목이 모두 입력되면 차량 진행상태가 **수출통관중** 으로 자동 전환됨.

### C-2. 수출통관 서류 발급 및 업로드

- [ ] **[통]** [서류] 탭 → **Proforma Invoice (PDF)** 다운로드 → 바이어에 송부.
- [ ] **[통]** [서류] 탭 → **Sales Contract (PDF)** 다운로드 → 바이어와 계약 진행.
- [ ] **[통]** 관세사·통관업체에 수출신고 의뢰. 면장(수출신고필증) 수령 시 **면장금액·면장번호** 입력.
- [ ] **[통]** [수출통관] 탭의 **수출통관 완료 체크박스** 체크 + **수출통관 문서(면장)** 업로드.

> **게이트 — 수출통관 mismatch 확인 모달**: 체크박스만 켜거나 면장 문서만 올린 상태로 저장하면 ⚠ **단계 진입 누락 확인** 모달이 뜸. 둘 다 처리해야 진행상태가 **수출통관완료** 로 넘어감.

- [ ] **[통]** 저장 후 차량 진행상태가 **수출통관완료** 인지 확인.

### C-3. 선적 정보 입력 — [선적] 탭

- [ ] **[통]** 차량 편집 패널 → [선적] 탭 열기.
- [ ] **[통]** 다음 **필수 항목** 입력:
  - B/L 바이어
  - B/L 컨사이니
  - B/L 반입지
  - 선박명 (VSL)
  - 컨테이너번호 (CONTAINER 선적 시)
- [ ] **[통]** **B/L 반입지** 가 입력되면 차량 진행상태가 **선적중** 으로 자동 전환.

### C-4. B/L 문서 업로드 및 전달

> **게이트 — 50% 룰 B/L 잠금 (큐 9 확장 — 구현 완료)**: B/L 문서 신규 업로드 시점에 미수율이 50%를 초과하면 시스템이 저장을 차단합니다 ("B/L 발행 차단" 토스트). 분기 3가지:
>
> 1. **정상 차단**: 미수율 > 50% → 잠금. 영업이 추가 입금 받아 미수율 50% 이하로 낮추거나, [관]에게 **50% 룰 예외** 승인 요청.
> 2. **환율 미입력 외화 차량**: 미수율 계산 불가 (분모 = null) → 별도 메시지로 차단. [재]/[영]이 환율 먼저 입력.
> 3. **기존 차량(grandfather)**: 이미 B/L 문서가 있던 차량은 모든 변경 통과 (2026-05-18 사용자 결정 — 운영중 차량 우회).

- [ ] **[통]** B/L 발행 후 **B/L번호·B/L 발행일** 입력.
- [ ] **[통]** [선적] 탭의 **B/L 문서 업로드** 영역에 B/L 스캔본 업로드. 잠금 발생 시 위 분기 처리.
- [ ] **[통]** [서류] 탭에서 선적 방법에 맞는 CIPL 엑셀 다운로드:
  - RORO 선적 → **RO_CIPL** (.xlsx)
  - CONTAINER 선적 → **con_CIPL** (.xlsx)
- [ ] **[통]** B/L 문서 + CIPL 엑셀을 **포워딩사 담당자에 전달** (현재 이메일 자동 발송은 미구현 — 수동 발송. 향후 SMTP 도입 시 자동화 예정).
- [ ] **[통]** 저장 후 차량 진행상태가 **선적완료** 인지 확인.

### C-5. DHL 발송 — [DHL] 탭

- [ ] **[통]** 차량 편집 패널 → [DHL] 탭 열기.
- [ ] **[통]** 다음 항목 입력:
  - 수취인명·수취인주소·수취인전화
  - 발송인명·발송인주소
  - 중량·크기
- [ ] **[통]** DHL을 통해 원본 서류 발송 후 **DHL 발송 신청 체크박스** 체크 → 저장.
- [ ] **[통]** 차량 진행상태가 **거래완료** 로 자동 전환되는지 확인.

## D. (부록) 정산 — 거래완료 후

> 사용자가 명시한 메인 흐름(영업→재무→통관→선적→B/L→끝)에는 포함되지 않지만, 시스템 운영상 거래완료 후 정산이 별도 단계로 존재합니다.

- [ ] **[재]** 사이드바 → **정산** → 우측 상단 **[+ 신규 등록]** 으로 대상 차량의 정산 행 **수동 생성**. (거래완료 자동 생성 아님)
- [ ] **[재]** 정산 마진 자동 계산값 확인:
  - 판매금원화 = (면장금액 − 운송비 USD) × 환율
  - 정산판매금원화 = 판매금원화 − 비용 9종 합계
  - 판매마진 = 정산판매금원화 − 매입가
  - 부가세마진 = 매입가 × 9%
  - 총마진 = 판매마진 + 부가세마진
- [ ] **[재]** 영업담당자별 정산비율·정산액·실지급액 확인 → **정산 확정**.
- [ ] **[재]** **정산 지급** 요청 → [관] 승인 큐로 이동.
- [ ] **[관]** 사이드바 → **승인 큐** → **정산 지급** 요청 확인 → 승인.
- [ ] **[재]** 승인 완료 후 실지급 처리 → 정산 상태 **지급완료** 로 전환.

## E. (부록) 공통 게이트 모음

처리 도중 마주칠 수 있는 시스템 잠금 / 확인 모달 목록입니다. 흐름 중간 어디서든 발생할 수 있습니다.

### E-1. 닫기 확인 모달 (큐 18)

차량·바이어·컨사이니·포워딩사 편집 슬라이드 패널에서 **변경사항을 저장하지 않고** 닫으려 하면 모달 표시.

- 문구: "변경 사항이 있습니다 / 저장하지 않고 닫으면 변경 내용이 사라집니다"
- 버튼: **[취소]** (편집 계속) / **[닫기]** (변경 폐기)

### E-2. 단계 진입 누락 확인 모달 (큐 21 후속)

말소 또는 수출통관에서 **체크박스와 문서 업로드 중 한쪽만** 처리한 채로 저장 시도하면 표시.

- 문구: "⚠ 단계 진입 누락 확인 — 아래 항목은 체크박스와 문서 업로드가 한쪽만 되어있어 해당 단계가 진행되지 않습니다"
- 처리: 안내 확인 후 양쪽 모두 처리하고 다시 저장.

### E-3. 관리자 승인 요청 (큐 14 G2)

다음 4가지 액션은 [관] 승인 없이는 진행 불가. **[승인 요청]** 버튼 클릭 후 [관] 사이드바 → **승인 큐** 에서 승인 처리.

| 요청 유형 | 트리거 | 누가 요청 | 누가 승인 |
|---|---|---|---|
| 같은 바이어 미수 + 신규 거래 | 판매 단계에서 미수 잔존 바이어 배정 시 | [영] | [관] |
| 50% 룰 예외 | B/L 문서 신규 업로드 시 미수율 50% 초과 | [통] | [관] |
| 자금이체 (잔금 취소 등) | 이미 확정된 잔금 되돌리기 | [재] / [영] | [관] |
| 정산 지급 | 정산 확정 후 실지급 처리 | [재] | [관] |

### E-4. 자금이체 5상태

[재] 사이드바 → **재무 처리** 에서 표시되는 자금이체 행 상태.

| 상태 | 표시 | 의미 |
|---|---|---|
| 대기 | 호박색 "대기" | 영업이 입력 + [관] 승인 완료, 재무 확정 대기 |
| 확정 완료 | 녹색 "확정" | 재무가 확정 처리 완료, 회계 반영됨 |
| 재무 거부 | 빨강 "재무 거부" | 재무가 거부, 영업 재입력 필요 |
| 취소 대기 | 빨강 ⊘ + "대기" | 취소 이체가 [관] 승인 후 재무 확정 대기 |
| 원상복구 완료 | 빨강 ⊘ + "확정" | 취소 이체 재무 확정, 원거래 무효 |

### E-5. 회계 무결성 잠금 (큐 20-D)

**확정** 상태로 전환된 잔금 행은 금액·날짜를 더 이상 수정·삭제할 수 없습니다. 변경 필요 시:

1. **[재/영]** 차량 편집 패널에서 취소 요청 생성.
2. **[관]** 승인 큐에서 승인.
3. **[재]** 재무 처리에서 취소 이체 확정.
4. 영업이 올바른 값으로 새 잔금 행 추가.
5. **[재]** 재무 확정.

```
[영] 차량 등록 (기본정보 입력)
  │
  ├─ [영] 매입 정보 입력 + 매입 잔금 추가 → "대기"
  │     └─ [재] 재무 처리 → 매입 잔금 확정 → "확정"  ★ 게이트 ★
  │
  ├─ [영] 말소 신청서 발급 (서류 탭) → 행정 처리 → 말소 체크 + 서류 업로드
  │     └─ 진행상태: 매입중 → 매입완료 → 말소완료
  │
  ├─ [영] 판매 정보 입력 (바이어·환율·판매가) + 판매 잔금 추가 → "대기"
  │     └─ [재] 재무 처리 → 판매 잔금 확정 → "확정"  ★ 게이트 ★
  │     └─ 진행상태: 판매중 → 판매완료
  │
  ├─ [통] 수출통관 정보 입력 (수출 바이어·포워딩사·선적일)
  │     └─ [통] 인보이스·계약서 PDF → 바이어 송부
  │     └─ [통] 면장 수령 → 체크박스 + 면장 업로드  ★ 게이트(mismatch) ★
  │     └─ 진행상태: 수출통관중 → 수출통관완료
  │
  ├─ [통] 선적 정보 입력 (B/L 바이어·반입지·선박명)
  │     └─ [통] B/L 발행 → B/L 문서 업로드  ★ 게이트(50% 룰 B/L 잠금) ★
  │     └─ [통] CIPL 엑셀(RO 또는 con) 다운 → 포워딩사 전달
  │     └─ 진행상태: 선적중 → 선적완료
  │
  ├─ [통] DHL 정보 입력 + 발송 → DHL 발송 신청 체크
  │     └─ 진행상태: 거래완료
  │
  └─ [재] (부록) 정산 메뉴에서 신규 정산 수동 생성 → 마진 확인 → 정산 지급 요청
        └─ [관] 승인 큐 → 정산 지급 승인  ★ 게이트 ★
        └─ [재] 실지급 처리 → 정산 지급완료
```
