## 한 줄 요약

보증금으로 매입 **계약금만** 대고, 바이어가 판매쪽에 **총판매가의 60%를 자기 돈으로** 넣어야 그 차가 선적으로 넘어간다. 기존 선적 게이트(미수율 50%)를 **"이 차만 60%(미수율 40%)"로 조인 것** + 알림 레이어.

## 배경 / 의도 (jin)

- 보증금(여력)을 매입 계약금으로 쓰되, **잔금은 바이어 돈 60%가 들어와야 진행**되게 하려는 것.
- 목적 = 바이어가 자기 보증금만으로 차를 무한정 사가는 걸 막고, **fresh cash로 커밋**을 증명하게.
- "결국 보증금을 계약금으로 쓰는데 잔금은 바이어 돈으로 60%를 입금해야 진행될 수 있도록."

## 왜 환산이 필요 없는가 (핵심 단순화)

- 바이어 입금은 **판매탭에 자기 통화(외화)**로 들어오고, **총판매가도 같은 외화** → 같은 통화끼리 60% 비교, 환산 0.
- 판매가 > 매입가(마진)라, 바이어가 판매쪽 60%를 현금으로 넣으면 그 돈이 매입 원가 잔액을 덮고도 남음 → "바이어 돈으로 매입한 셈" 성립.
- C2 선지급은 **매입 PBP(원화)**라 판매 입금엔 안 잡힘 → 차의 판매 입금은 전부 "바이어 fresh cash"로 자연 계산.

## 자연 해소 흐름 (매입 미납이 저절로 풀리는 경로)

1. 보증금 → 계약금 → 차 확보 (**매입중**, 잔금 남음, [!매입] 표시 + '보증금 매입' 뱃지)
2. 바이어가 판매쪽 현금 60%+ 입금 → **선적 락 풀림**
3. **재무가 그 현금으로 셀러에게 매입잔금 지급** → PBP 기록 → **매입 GREEN**
4. 말소·수출·선적 진행

→ [!매입]은 3번에서 자연 소멸. "바이어 돈으로 매입"의 실제 구현 = "바이어 판매입금 → 재무가 그 돈으로 매입잔금 지급".

## 기각된 대안

- **락 타이밍 (A) 10일 후 발동**: 3일차에 60% 안 냈어도 선적 시도 가능한 구멍. (B) 상시로 대체.
- **60% 측정 = 매입잔금 KRW 환산 대조**: 통화·주체 안 맞고 환율 스냅샷 필요. 판매쪽 외화 60%로 단순화(환산 0).
- **매입 미납 = 바이어 완납 시 자동 GREEN**: 실제 셀러 미지급을 숨겨 말소·수출 title 블로커를 놓칠 위험. 맥락 분리 마커로 대체.
- **60% 미달 시 선지급 롤백**: void 미지원. 진행 차단으로 대체.

## 구현 시 아직 설계할 것 (착수 전 결정)

1. **'보증금 매입' 마커 감지법** — C2 PBP엔 "보증금 선지급" 마커 존재. 이번 부분지급 PBP는 계약금분만. 차량 단위 플래그를 새로 둘지, "보증금 선지급 마커 PBP 有 + 매입 미지급 有"로 판정할지.
2. **알림톡 신규 템플릿** — "바이어 60% 판매입금 독촉"은 기존 14종에 없음 → 새 템플릿 승인(리드타임). 승인 전엔 **deploy-inert**(게이트만 먼저, 알림 뒤).
3. **C2 부분지급 모드** — 현 매입 전액 완납 로직에 계약금만 넣는 모드 추가 (기존 전액 모드 보존).
