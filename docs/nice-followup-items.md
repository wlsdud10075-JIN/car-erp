# NICE 연동 — 사용자 확인/조사 사항 (2026-05-24)

NICE 연동(ssancar-erp 미들웨어 경유)은 완료됐고, **표준 필드(VIN·차명·소유자·주행거리·배기량 등)는 저장·문서 로드까지 정상**이다.
아래 2건만 이번 매핑표에서 빠져 미구현 상태 — **사용자가 실 NICE 응답을 확인한 뒤 결정**한다.

> 참고: 미매핑 NICE 필드도 응답 원본 전체가 `vehicles.nice_raw`(JSON)에 보존된다(조회 후 저장 시).
> 통관 SET 서류는 이미 `nice_raw`에서 형식·제원관리번호·출력·검사기간을 자동 로드한다.

---

## 1. 기통수 (통관 SET 구매리스트 G12 "기통")

**현재 상태**: 공란.

- NICE `engineSpec` 필드가 `"6/3342"` 형태(= **기통수 / 배기량**)로 `nice_raw`에 보존돼 있음.
- 이번 구현은 `engineSpec`에서 **배기량(`/` 뒤 숫자)만** 추출 → 배기량/cc 칸은 자동.
- **기통수(`/` 앞 숫자)는 추출하지 않음.** 또한 통관 서류 G12는 `nice_spec_cylinders` **컬럼**을 읽는데 그 컬럼이 안 채워짐.

**확인할 것**:
- 실제 NICE `engineSpec` 응답이 정말 `"기통수/배기량"` 순서인지 (예: `"6/3342"`).
- 기통수를 서류에 자동으로 넣을지 여부.

**확정 시 작업(빠름)**: 통관 G12를 `nice_raw`의 `engineSpec` 앞 숫자에서 뽑도록 `DocValue` 헬퍼로 변경(컬럼 채우기 불필요) — 약 10분.

---

## 2. 검사 종료일 (통관 SET 구매리스트 I11 "종료")

**현재 상태**: 공란. (검사 시작 I10에는 `resValidPeriod` 전체가 들어감)

- NICE `resValidPeriod`(검사 유효기간)가 보통 **"시작~종료" 범위 문자열**로 옴.
- 이번 구현은 분할 없이 `resValidPeriod`를 I10(검사시작)에 통째로 기입, I11(검사종료)은 공란.

**확인할 것 (가장 중요)**:
- `resValidPeriod` 실제 형식. 예:
  - `"20250428~20270427"` (구분자 `~`, YYYYMMDD)
  - `"2025-04-28 ~ 2027-04-27"` (구분자 ` ~ `, Y-m-d)
  - 혹은 시작/종료가 **애초에 별도 필드**로 오는지 (예: `resValidPeriodStart`/`End`).

**확정 시 작업**: 형식 확인되면 시작/종료로 분할 → I10/I11 각각 기입. 형식만 알면 약 15분.

## 다음 행동
- 위 1·2의 "확인할 것"을 **실제 ssancar/NICE 응답 1건**으로 확인.
- `nice_raw`에 응답 원본이 통째로 저장되므로, 차량 1대 조회·저장 후 그 차량의 `nice_raw`(JSON)를 열어보면 `engineSpec`·`resValidPeriod` 실제 값을 바로 확인 가능.
- 확인되면 알려주면 두 칸 모두 마감.
