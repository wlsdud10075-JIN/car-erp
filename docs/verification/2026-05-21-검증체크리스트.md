## A. 정산 안건 (#2 시리즈) — 핵심 운영 흐름

### A-1. Salesman.type 설정 (사전 준비 필수)
1. `/erp/salesmen` → 기존 영업담당자 1명 편집 → **"정산 분류"** select → "프리랜서" 또는 "사내직원" 선택 → 저장
2. 다른 담당자도 다른 type으로 설정 (검증 비교용)
3. ⚠️ 시드 데이터의 영업담당자는 모두 default `employee` (사내직원)

### A-2. 거래완료 시 자동 정산 생성
1. A-1에서 type 설정한 영업담당자의 차량 1대 → B/L 문서 업로드 → 진행상태 **"거래완료"**
2. `/erp/settlements` → 그 차량 정산 row 자동 생성 (status=대기)
3. **프리랜서** 담당이면 정산방식 **"비율"** / **사내직원**이면 **"건당"**
4. ratio/amount 비어있음 — 재무가 채우는 흐름 (의도)

### A-3. 정산 영업담당자별 합계 카드
1. `/erp/settlements` 필터 바 아래 → 담당자별 카드 그리드
2. 카드에 정산건수·총마진·정산액·**실지급액 합계**
3. 카드 클릭 → 해당 담당자만 솔팅 토글 (재클릭으로 해제)

### A-4. 입금률 게이지
1. 정산 표의 **"입금률"** 컬럼
2. 완납 차량 → 초록 100% bar / 부분 입금 → amber bar + **"받을 ₩X"** 텍스트
3. 환율 미입력 외화 → "환율 미입력" 회색

### A-5. 재무 대시보드 — 정산 차단 KPI
1. 재무 role 또는 admin 로그인 → `/erp/dashboard`
2. `[역할별] → [재무]` 토글 → KPI 5번째 카드 **"정산 차단 (거래완료 미수)"** 노출
3. 거래완료지만 미수금 남은 차량 수 + 총 미수금 hint

## B. 수출통관 안건 (#1)

### B-1. 사이드바 메뉴
1. **수출통관** role 또는 admin 계정 → 사이드바 "수출통관" 클릭
2. 차량 목록에 **통관 후보 합집합** 노출:
   - (a) 매입완료 + 판매 진행 + **말소 안 됨** (말소 푸시 대상)
   - (b) 말소완료 + 판매 진행 + **입금 50% 이상** (통관 진행 가능)

### B-2. 말소 대기 KPI 카드
1. 수출통관 role 또는 admin이 `[역할별] [수출통관]` 토글
2. KPI 첫 번째 카드 **"말소 대기"** 표시 (총 5개 카드 그리드)
3. 카드 클릭 → 차량 목록의 `deregistration_needed` 액션으로 이동
