# [관리] 시선 워크플로우 체크리스트 — 매입 → 거래완료

> 작성일: 2026-05-26 · 대상: `role=관리` (본인 담당 영업 5명 관할)
> 검증 근거: `tests/Feature/ManagementWorkflowChecklistTest.php` (12 테스트 / 40 assertions, 전부 통과)
> 기준 구현: 현재 `dev` 브랜치. 각 행은 실제 코드 동작을 실행 검증한 결과.

이 문서는 [관리]가 영업 5명이 올리는 차량을 **매입부터 거래완료까지** 단계별로 점검할 때 쓰는 체크리스트다.
각 단계에는 **락(게이트)** 이 걸려 있고, 락을 통과하지 못하면(fail) 저장이 차단된다. 아래 표에 **fail 사유 메시지**와 **통과(해결) 방법**을 1:1로 정리했다.

## 1. 워크플로우 10단계 + 책임 role (v4)

진행상태는 DB 컬럼이 아니라 computed (`Vehicle::progress_status`). 우선순위 평가 후 첫 매칭 반환.

| 단계 | 진입 조건 | 주 책임 |
|---|---|---|
| **매입중** | 매입가 입력 | 영업 |
| **매입완료** | 매입 미지급 ≤ 0 (매입 잔금 confirmed) | 영업 + 재무 |
| **말소완료** | `is_deregistered` + 말소서류 | 영업 |
| **판매중** | 판매가 입력 | 영업 |
| **판매완료** | 판매 미입금 ≤ 0 | 영업 + 재무 |
| **선적중** | 반입지(`bl_loading_location`) 입력 | 수출통관 |
| **선적완료** | 반입지 + 수출신고서 | 수출통관 |
| **통관중** | 통관완료(`is_export_cleared`) + 반입지 | 수출통관 |
| **통관완료** | (실질 도달 불가 — 거래완료 우선) | 수출통관 |
| **거래완료** | B/L 문서 발급 | 재무 |

> v4 순서: 매입 → 판매 → **반입(선적) → 통관** → B/L → 거래완료.

## 2. 락(게이트) 전체 지도 — 9종

락은 **발동 위치**에 따라 두 부류다. 이걸 모르면 "왜 어떤 락은 안 걸리지?"를 답할 수 없다.

| 발동 위치 | 락 | 발동 조건 |
|---|---|---|
| **saving 훅 (자동)** | G2 / G1 / Ledger / 삭제가드 | 로그인 사용자가 모델 저장 시 자동. **시드·artisan(비로그인)·raw 스크립트는 우회.** |
| **UI save() (명시)** | C4 / 컨사이니 / C5 / H3 / H1 | 차량 편집 화면 저장 버튼 흐름에서만. **`Vehicle::create()`/`update()` raw 호출은 우회.** |
| **폼 검증** | H10 | 차량 편집 화면 validation. |

> ⚠️ **gotcha (관리 필독)**: `tinker`나 시더로 `Vehicle::create([...나쁜 데이터...])`를 하면 C4·C5·컨사이니·H3·H1이 **걸리지 않는다** (UI 가드라서). 운영 데이터를 손으로 만질 때 단계 건너뛰기가 가능하므로 raw 수정은 피한다.

### 락 요약표 (fail 사유 ↔ 해결법)

| # | 락 | trigger | fail 메시지(요약) | 해결법 | 대응 test |
|---|---|---|---|---|---|
| 1 | **G2** 동일바이어 미수 | 같은 바이어에 미수 잔존 차량 있는데 **신규 거래** 등록 | "이 바이어는 미수 잔존 차량이 있습니다. 신규 거래는 관리자 승인이 필요" | 관리가 **신규거래 승인**(차량번호 바인딩) 발급 → 1회 소진 | `test_lock_g2_*` |
| 2 | **G1** 50% B/L | B/L 문서 **신규** 첨부 + 미수율 > 50% | "B/L 발행 차단 — 미수율 X% (50% 초과)" | 잔금 **50% 이상 입금** / admin 미입금 우회(선적) / 기존 B/L 차량은 우회 | `test_lock_g1_*` |
| 3 | **Ledger** 회계필드 잠금 | 재무 확정 잔금 있는 차량의 매입가·판매가·환율·비용·바이어·담당자 변경 | "재무 확정 잔금이 있는 차량의 회계 영향 필드는 잠금 해제 후 수정" | admin/super가 **[🔓 잠금 해제]** + 사유 10자↑ → **1회** 변경 후 자동 재잠금 | `test_lock_ledger_*` |
| 4 | **삭제가드** | 확정 잔금 차량을 **비-admin**이 삭제 | "재무 확정 잔금이 있는 차량은 admin/super만 삭제" | admin/super가 삭제 | `test_lock_delete_*` |
| 5 | **C4** 말소완료 강제 | 말소 미완료 상태로 **통관 정보 입력** | "말소 처리(체크 + 서류 업로드)를 완료한 후 통관 진입" | `is_deregistered` 체크 + 말소서류 업로드 | `test_lock_c4_*` |
| 6 | **컨사이니** | 반입지 입력하는데 **판매 컨사이니 미지정** | "선적 진입 전 판매 컨사이니를 지정해야 합니다 (판매 단계)" | 판매 탭에서 `consignee_id` 지정 | `test_lock_consignee_*` |
| 7 | **C5** 50% 통관 | 판매가 있고 입금률 < 50%로 통관/선적/DHL 진입 | "판매 입금률 < 50% ... 단계 진입 불가" | 잔금 **50% 이상 입금** / admin 미입금 우회 | `test_lock_c5_fifty_*` |
| 7b | **C5(외화)** 환율 미입력 | 외화 차량 + 환율 0으로 통관 진입 | "환율 미입력 외화 차량은 통관 진입 불가" | **환율 입력** / admin 미입금 우회 | `test_lock_c5_foreign_*` |
| 8 | **H3** B/L→반입지 | 반입지 없이 B/L 문서 업로드 | "B/L 문서를 업로드하려면 선적 반입지 입력이 먼저 필요" | **반입지 먼저 입력** | `test_lock_h3_*` |
| 9 | **H1** DHL→B/L | B/L 문서 없이 DHL 발송 신청 | "DHL 발송 신청을 하려면 B/L 문서 업로드가 먼저 필요" | **B/L 문서 먼저 업로드** | `test_lock_h1_*` |

> ⚠️ **발동 순서 (한 번에 하나만 보임)**: 한 저장에 여러 위반이 겹치면 **첫 번째** 락만 메시지로 뜬다.
> - saving 훅 순서: `Ledger → G1 → G2`
> - UI save() 순서: `C4 → 컨사이니 → C5 → H3 → H1 → H10`
> - 예: 말소 미완료 + 컨사이니 미지정 + 미수 100%로 한 번에 저장하면 **C4 메시지**만 보인다. C4 해결 후 다시 저장하면 그제서야 컨사이니 메시지가 뜬다.

## 3. 단계별 점검 체크리스트

각 단계에서 [관리]가 영업에게 확인시킬 항목과, 거기서 만날 락.

### ① 매입 (영업)
- [ ] 매입가(`purchase_price`) 입력됨 → **매입중** 진입
- [ ] 매입 잔금이 **재무 확정(confirmed)** 되어 미지급 0 → **매입완료**
- [ ] ⚠️ 확정 잔금이 생기는 순간부터 **Ledger 락(#3)** 활성 — 이후 매입가/판매가/비용/바이어/담당자 변경은 admin 잠금 해제 필요

### ② 말소 (영업)
- [ ] `is_deregistered` 체크 + 말소서류 업로드 → **말소완료**
- [ ] ⚠️ 말소 미완료면 이후 **통관 진입이 C4 락(#5)** 으로 차단됨
- [ ] ⚠️ 말소 처리 시 소유자 **주민(법인)등록번호 필수** (H10 폼검증) — 말소/양도 서류가 RRN 사용

### ③ 판매 (영업 + 재무)
- [ ] 판매가(`sale_price`) 입력 → **판매중**
- [ ] **판매 바이어** 지정 — 같은 바이어에 미수 잔존 차량 있으면 **G2 락(#1)**, 관리 승인 필요
- [ ] **판매 컨사이니** 지정 — 안 하면 나중에 선적 진입이 **컨사이니 락(#6)** 으로 막힘
- [ ] 외화면 **환율 입력** — 0이면 통관/선적이 **C5(외화) 락(#7b)** 으로 막힘
- [ ] 판매 잔금 확정 입금률 챙기기 — 50% 미만이면 통관·B/L이 **C5(#7)/G1(#2)** 으로 막힘 → **판매 미입금 0이면 판매완료**

### ④ 통관 (수출통관)
- [ ] (선행) 말소완료 — C4 통과
- [ ] (선행) 입금률 ≥ 50% 또는 admin 우회 — C5 통과
- [ ] 통관 바이어/포워딩사/수출신고서 입력

### ⑤ 선적 (수출통관)
- [ ] (선행) 판매 컨사이니 지정 — 컨사이니 락 통과
- [ ] 반입지(`bl_loading_location`) 입력 → **선적중**

### ⑥ B/L (재무/수출통관)
- [ ] (선행) 반입지 입력됨 — **H3 락(#8)** 통과
- [ ] (선행) 입금률 ≥ 50% 또는 admin 우회 — **G1 락(#2)** 통과
- [ ] B/L 문서 발급 → **거래완료**

### ⑦ DHL (선택)
- [ ] (선행) B/L 문서 업로드됨 — **H1 락(#9)** 통과
- [ ] DHL 발송 신청

### ⑧ 거래완료 후 (재무)
- [ ] 거래완료 진입 시 정산(Settlement) 자동 생성 (프리랜서 ratio 50% / 사내직원 per_unit 10만 기본)
- [ ] ⚠️ 확정 잔금 차량 삭제는 **admin/super만** (삭제가드 #4)

## 4. 락 통과(해결) 실전 가이드

테스트로 검증된 정확한 해결 절차.

### G2 — 동일 바이어 미수 + 신규 거래
- **왜 막히나**: 같은 바이어가 이미 미수금이 있는데 또 신규 차량을 거래로 등록하면 미수 누적 위험. 영업 단독 진행 차단.
- **통과**: 영업이 [신규 거래 승인 요청] → **관리가 승인**. 승인은 `payload.new_vehicle_number`에 **차량번호 바인딩**(1 승인 = 1 차량). 다른 차량번호로 쓰면 거부. 승인은 저장 시 1회 소진(`used_at`).
- **관리 actor**: `canApprove()`(admin/super/관리)는 G2 **자동 우회** — 관리가 직접 등록하면 안 걸림.

### G1 — 50% B/L 잠금
- **왜 막히나**: 미수율 50% 초과 차량은 B/L(선하증권) = 화물 인도권을 넘기면 떼일 위험. 발행 차단.
- **통과**: ① 잔금 50% 이상 입금(미수율 ≤ 50%) / ② admin **미입금 우회 승인(선적 단계)** / ③ 이미 B/L 있던 차량은 grandfather 우회.
- ⚠️ **관리는 미입금 우회 승인 발급 불가** — `canApproveUnpaidExport()`는 admin/super 전용(`canApprove()`와 별개). 우회가 필요하면 관리는 admin/super로 **에스컬레이션**.

### Ledger — 회계 영향 필드 잠금
- **왜 막히나**: 재무가 잔금을 확정한 뒤 매입가·판매가·환율을 소급 변경하면 정산·전표가 어긋남. 소급 변경 차단.
- **통과**: admin/super가 `VehicleLedgerUnlockService::unlock()` ([🔓 잠금 해제] + **사유 10자 이상**) → **딱 1회** 변경 가능 → 저장 직후 자동 재잠금. 추가 변경하려면 다시 해제.
- 빈 값 → 최초 입력은 잠금 대상 아님(정상 흐름 보호). 이미 값이 있던 필드의 변경만 차단.

### C4 — 말소 완료 강제
- **통과**: `is_deregistered` 체크 + 말소서류 업로드 후 통관 정보 입력.

### 컨사이니 — 선적 전 판매 컨사이니
- **통과**: 판매 탭에서 `consignee_id` 지정 후 반입지 입력.

### C5 — 50% 통관 게이트
- **통과**: 잔금 50% 이상 입금 / admin 미입금 우회 승인(해당 단계). 외화 환율 미입력이면 **환율 입력** 또는 우회 승인.
- ⚠️ G1과 동일 — **관리는 우회 승인 발급 불가**(admin/super 전용). 관리는 환율 입력/입금 유도로 자력 통과시키거나 admin 에스컬레이션.

### H3 / H1 — 첨부 선행
- **통과**: 반입지 → B/L 문서 → DHL 신청 **순서**대로. 건너뛰면 차단.

## 5. 채널 차이 (export 외)

본 체크리스트는 **export(수출)** 풀흐름 기준. 국내 채널은 단순하다.

| 채널 | 흐름 | 락 적용 |
|---|---|---|
| `export` 수출 | 매입→말소→판매→통관→선적→B/L→DHL→거래완료 | 락 9종 전부 |
| `heyman` 헤이맨 | 국내 바이어, 원화 정산 | 통관/선적/B/L 단계 없음 → C4·C5·컨사이니·H3·H1 비해당. G2·Ledger·삭제가드는 동일 적용 |
| `carpul` 카풀 | 국내 바이어, 원화+VAT, 대행수수료 | heyman과 유사 + 계산서 필드. 락 적용 동일 |

> G2(미수 신규거래)·Ledger(회계 잠금)·삭제가드는 채널 무관 적용.
