재무 업무 가이드재무

판매 입금 → 매입 지급 → 미수금 관리 → 정산 확정 → 2차 정산
바이어 입금 확인·입력, 매입처 지급 처리, 미수금 관리, 그리고 거래완료 차량의 정산 확정까지 돈과 관련된 전 과정을 담당합니다. car-erp의 판매·매입 탭과 정산 화면에서 진행합니다.
※ 핵심 원칙 — 입금/지급/정산은 재무가 확정까지, 실제 지급(paid)은 [관리]가 승인 (직무 분리). 현재는 [관리]가 이 업무도 겸함.

📋 단계별 상세

1
판매 입금 입력
입금 확인 즉시
할 일 — 판매 탭 입금현황
  1. 바이어 송금을 확인하면 그 금액을 입력 — 계약금 / 중도금 / 잔금(N건) / 선수금.
  2. 외화 차량은 통화·환율을 반드시 같이 입력 (환율 0이면 미수금 계산이 깨짐).
  3. 입력하면 입금 상태가 자동 갱신 — 완납 / 부분입금(%) / 미입금.
💡 입금액 입력 = 곧 확정 처리입니다. 입금률이 50%를 넘으면 통관·선적이, 100% 완납되면 B/L 발급이 가능해집니다(수출통관 가이드와 연결).
⚠️ 환율 필수
외화차에 환율을 안 넣으면 미수금이 0으로 잘못 잡혀 "완납"으로 오인됩니다. 판매가 입력 시 통화·환율·판매일·바이어 항상 함께.
2
매입 지급 처리
매입 후
할 일 — 매입 탭
  1. 매입가·매도비 확인, 계약금/잔금(N건) 지급 입력.
  2. 지급일이 도래한 건만 미지급에서 차감됩니다 (예정일 미래 건은 제외).
  3. 매입처 계좌 등 민감정보는 권한 범위 내에서만.
3
미수금 · 채권 관리
상시
할 일 — 채권관리 / 대시보드
  1. 미입금·부분입금 차량을 모니터링 (담당자별·바이어별 TOP10 미수금).
  2. 환율 미입력 외화 차량은 별도 경고로 분리 표시 → 환율 채우기.
  3. 회수 진행 상황을 입금현황에 반영.
4
정산 확정
거래완료 시
정산 상태 흐름
pending자동 생성 → confirmed재무 확정 → paid[관리] 지급
  1. 거래완료되면 정산이 pending으로 자동 생성 (마진·정산액 자동 계산).
  2. 금액 검토 후 confirmed(확정) 처리 — 재무 담당.
🔐 직무 분리 (중요)
재무는 paid(지급)를 직접 못 합니다. 확정(confirmed)까지가 재무, 실제 지급(paid)은 [관리]/관리자 승인으로만. 본인이 요청하고 본인이 지급하는 것 차단.
5
2차 정산 (사후 보정)
지급 후 ~1개월
할 일
  1. 지급(paid) 후 한 달간 실측 비용 보정 — 말소비·탁송·쇼링·보험 등 9개 비용 실제값 수정.
  2. 정산 시점 환율 정정(환차 반영).
  3. "2차 완료" 처리 → closed(회계 잠금). 환차·이월이 자동 계산됩니다.
⚠️ 잠금 주의
confirmed/paid 이후 회계 데이터는 함부로 못 바꿉니다(스냅샷 보존). 비용 보정은 2차 정산 절차 안에서만.

⚠️ 자주 하는 실수

  1. 외화 환율 미입력 → 미수금이 0으로 잘못 잡혀 완납 오판.
  2. 재무가 paid 직접 시도 → 불가([관리] 승인).
  3. 지급일이 미래인 매입 잔금을 미지급으로 오인.
  4. 확정 후 회계 컬럼 임의 수정 → 2차 정산 절차로만.
