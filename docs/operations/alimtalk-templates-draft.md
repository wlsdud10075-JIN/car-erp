# 카카오 알림톡 템플릿 문구 초안 (11종)

> car-erp(중고차 수출 ERP) → BizM/스윗트래커 카카오 알림톡 발송용 템플릿 문구 초안.
> 대표(최고관리자, admin) 5종 + [관리] role 6종.

---

> ⚠️ **코드 구현 반영 (2026-07-04)** — 현재 ERP는 발송 시 **본문(msg) + 강조 title 만** 보낸다(버튼 URL은 payload로 안 넘김).
> - **버튼은 선택**이다. 안 넣어도 알림은 정상 발송된다(문구가 "ERP에서 확인해 주세요"로 자기완결). **첫 등록은 버튼 없이** 하는 걸 권장 — 승인이 빠르고 WL 버튼 도메인 사전등록 이슈를 피한다. 나중에 원하면 각 템플릿에 고정 ERP URL 버튼(예: `https://heysellcar.com/erp/settlements`)을 추가하면 BizM이 자동으로 붙여 보낸다(코드 수정 불필요).
> - **12번 말소등록증전달은 버튼 금지** — 링크가 차량마다 달라져서(만료 서명 링크) 고정 버튼으로 못 만든다. 그래서 **본문에 `#{링크}` 변수**로 넣었다. `#{링크}`는 그대로 두면 ERP가 발송 시점에 채운다.

## 📌 사용법 (jin — BizM 등록 순서)

1. BizM 콘솔 → **[알림톡 템플릿]** 메뉴에서 아래 12종을 **하나씩 신규 등록**한다.
   - **템플릿 코드**: 각 항목의 `템플릿 코드`(영문)를 **그대로** 입력 (`erp_` 접두). ⇒ 이 코드를 ERP 기능설정의 "템플릿ID" 칸에 **똑같이** 넣어 1:1 매칭한다.
   - **강조유형**: 아래 표기(기본형/강조표기형)대로 선택. 강조표기형이면 상단 **강조 제목(title)** 도 함께 입력.
   - **본문**: 아래 "본문 문구" 를 **그대로 복붙**. `#{변수명}` 은 BizM 이 변수 자리로 자동 인식한다.
   - **버튼**: 위 상단 메모 참고 — 첫 등록은 생략 권장, 12번은 반드시 생략.
2. 등록 후 각 템플릿을 **검수 요청** → 카카오 승인(보통 1~2 영업일).
3. 승인되면 **템플릿 코드 12개 + userid + 발신프로필키**를 ERP **기능설정 → 알림톡** 에 입력한다.

## ⚠️ 주의

- **발신프로필 1개**에 아래 12종(대표5+관리6+바이어1)을 모두 등록한다. (car-erp 전용 발신프로필)
- 기존에 쓰던 다른 알림톡 템플릿과는 **별개** — 반드시 `erp_` 접두 코드로 구분해 혼선을 막는다.
- **정보성 메시지만 승인됨.** "지금 확인하세요!", "놓치지 마세요", 할인·프로모션·이모지 과다 등 **광고성/과장 표현은 반려**된다. 아래 문구는 전부 담백한 사실 통보형으로 작성됨 — 문구를 임의로 "홍보체"로 바꾸지 말 것.
- 회사명은 발신 채널(카톡 상단 채널명)에 자동 노출되므로 본문에 굳이 안 넣어도 된다. 넣고 싶으면 `#{회사}` 변수로.
- 금액 변수는 ERP 에서 콤마 포함 문자열(예: `12,500,000원`)로 치환해 넣는 것을 권장(알림톡은 숫자 서식을 지원 안 함).

## 1. 거래완료

| 항목 | 내용 |
|---|---|
| **알림 이름** | 거래완료 |
| **수신자** | 대표(admin) |
| **트리거 시점** | 차량에 B/L 문서 발급 → 진행상태 `거래완료` 진입 시 |
| **템플릿 코드** | `erp_deal_done` |
| **강조유형** | 강조표기형 (title: `거래완료`) |

**본문 문구**
```
[거래완료] #{차량번호}

차량 한 대의 거래가 완료되었습니다.

▶ 차량번호: #{차량번호}
▶ 바이어: #{바이어}
▶ 판매금액: #{판매금액}
▶ 마진: #{마진}

B/L 발급으로 딜이 종료되었습니다.
```

**변수 목록**
- `#{차량번호}` : 차량 번호판 (예: 19더9065)
- `#{바이어}` : 판매 바이어명
- `#{판매금액}` : 판매가(통화 포함, 예: USD 12,500)
- `#{마진}` : 총마진(원화, 예: 3,200,000원)

**버튼**: `차량 보기` (웹링크 WL) → `{{링크}}` (ERP 차량 상세)

**등록 주의**: 실적 통보(정보성). 마진 수치는 사실 통보이므로 문제없음 — "역대급 마진" 등 수식어만 금지.

## 12. 말소등록증전달

| 항목 | 내용 |
|---|---|
| **알림 이름** | 말소등록증전달 |
| **수신자** | **국내 바이어** (직원 아님 — 판매 바이어 전화번호) |
| **트리거 시점** | 말소등록증 업로드 후, 매입 탭에서 담당자가 **수동 버튼**으로 발송 |
| **템플릿 코드** | `erp_deregistration_notice` |
| **강조유형** | 기본형 |

**본문 문구**
```
[말소등록증 발급 안내] #{차량번호}

구매하신 차량의 자동차 말소등록증이 발급되었습니다.

▶ 차량번호: #{차량번호}

아래 링크에서 말소등록증을 확인·다운로드하실 수 있습니다.
#{링크}

(보안을 위해 링크는 3일 후 만료됩니다.)
```

**변수 목록**
- `#{차량번호}` : 차량 번호판
- `#{링크}` : 말소등록증 열람 링크 (ERP가 발송 시점에 만료 서명 링크를 자동 생성해 채움)

**등록 주의**:
- 알림톡은 파일 첨부가 안 되므로 **본문에 링크**를 넣는 방식(WL 버튼 아님). `#{링크}` 변수는 그대로 두면 ERP가 채웁니다.
- 수신자가 국내 바이어라 문구는 존댓말 안내체. 광고성 표현 금지.
- 링크 도메인은 ERP 도메인(예: `heysellcar.com`)입니다. BizM이 링크 도메인 사전등록을 요구하면 해당 도메인을 등록하세요.
