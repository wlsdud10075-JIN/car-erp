# [관리] 운영 체크리스트

차량 1대를 매입부터 정산까지 책임지고 끌고 가는 흐름. 막히는 지점마다 해제 방법 함께.

## 0. 처음 한 번만 — 사전 준비

### 0-1. 본인 계정 확인
- [ ] 본인 계정 권한 — 역할이 **"관리"** 인지 확인 (사용자 관리 메뉴)

### 0-2. 영업담당자(휘하 직원) 등록 — 사용자 관리 메뉴

내 휘하 영업담당자를 한 명씩 등록.

![사용자 추가 화면](screenshots/00-user-add.png)

- [ ] **이름·이메일·전화·비밀번호** 입력
- [ ] **권한** = 일반사용자(user)
- [ ] **역할** = 영업 (또는 수출통관·재무 등)
- [ ] **정산 분류** 필수 — 사내직원(건당 정산) vs 프리랜서(비율 정산)
  ![정산 분류 선택](screenshots/00-user-settlement-type.png)
- [ ] **담당 [관리]** = 본인 선택 → 이 직원이 내 휘하로 등록됨
  ![담당 관리자 지정](screenshots/00-user-manager-assign.png)

> ⚠️ 정산 분류 누락 시 거래완료 차량 정산액이 0으로 자동 처리됩니다. 반드시 명시.

### 0-3. 항구 등록 — 항구 마스터 메뉴

![항구 마스터](screenshots/00-ports-master.png)

- [ ] **반입지 (한국 부두)** — HJIT / HPNT / ICT / IPFC / PCTC 등 부두 코드

> 통관 단계에서 드롭다운으로 사용. 없는 항구는 "+ 항구 추가" 슬라이드 패널에서 등록 가능.

## 한 대 처리 흐름 (1~8번 순서대로)

> 📌 **워크플로우 순서**: 매입 → 말소 → 판매 → **반입(선적) → 통관 → B/L** → DHL → 거래완료 → 정산
> ※ "선적"의 의미는 **반입지 입력**. 실제 운영 순서는 차량 반입 먼저, 그 다음 통관, 마지막에 B/L 발급.

### 3️⃣ 판매

![판매 탭 전체](screenshots/03-sales-tab.png)

#### 판매 기본

- [ ] **판매일·통화·환율·바이어·컨사이니** 선택
- [ ] 바이어가 없으면 **+ 신규** 버튼으로 바로 등록 가능
- [ ] 판매가·**TAX D/C**(세금 할인)·**Commission**(중개수수료)·운임비·자동하역비·기타 판매비용 입력

#### 입금 현황 — 4 항목 (재무·관리자만 입력 가능)

![입금 현황 그리드](screenshots/03-sales-payment-grid.png)

- [ ] **계약금 입금·중도금·선수금1** — 받은 만큼 입력
- [ ] **🆕 송금 수수료 (셀러 부담)** — 위치: 입금 현황 그리드 두 번째 줄 (계약금 입금 바로 아래)
  - 예: USD 5,000 송금 → 4,990 도착 + 10 수수료 → **계약금 4,990 + 송금 수수료 10**
  - 셀러(우리) 부담 → 미수금 자동 차감 (바이어는 5,000 다 보낸 것으로 인정)
  - 저장 시 환율 자동 snapshot → 누적 KRW 환산 자동
- [ ] **적립금 사용·적립금 적립** — 바이어 적립금에서 차감 또는 신규 적립
- [ ] **잔금** — 영업이 + 추가로 Draft 입력, 재무가 transfers에서 확정

#### 저장 후 — 4 항목 잠금 표시 + 바이어 카드 확인

![수수료 저장 후 lock 표시](screenshots/03-sales-fee-locked.png)

저장하면 **"재무 확정 잔금 있음 — 회계 영향 컬럼 잠김"** 표시되면서 매입가·판매가·환율·면장금액·비용9개·바이어·담당자 변경 불가 잠금 (관리/관리자가 사유 입력 후 잠금 해제 가능).

이제 **바이어 관리 → 그 바이어 클릭** → 패널 상단 보라색 카드 확인:

![바이어 누적 송금 수수료 카드](screenshots/03-buyer-fees-card.png)

> 💡 **영업 협상 카드**: 카드의 ₩ 금액 = 우리가 그 바이어를 위해 누적으로 부담한 송금 수수료. "그동안 N건, ₩XX,XXX 부담했습니다" 라며 협상 자료로 활용.

> 🔒 막힘: **"확정된 입금은 수정 불가"** 메시지
> → 재무에게 "이 입금 건 확정 취소해 주세요" 요청 → 수정 → 재무가 다시 확정

> 💡 송금 수수료 누락 시: 미수금이 6 USD 같이 어중간하게 남으면 십중팔구 수수료 입력 누락. 수수료 칸에 적어서 0으로 맞추기

### 6️⃣ B/L (선적 — 2차)

선적(B/L) 탭으로 다시 돌아가서:

- [ ] **B/L 번호·B/L 발행일** 입력
- [ ] **B/L 문서 업로드** ("업로드 시 선적완료 상태 달성 가능" 안내 옆 파일 선택)

> 🔒 막힘: **"잔금 미입금 — B/L 발급 불가"** (잔금 100% 완납 필수)
> → **통관(50%)과 다름. B/L은 100% 받아야 함.**
> → 해제 방법:
> - (a) 잔금 100% 받기
> - (b) **재무처리 메뉴 → "수출입금 게이트 우회"** → 차량 선택 → 사유 명시 → 승인 (관리/관리자만 가능, 통관 우회와 **별개로 한 번 더 발급 필요**)

### 8️⃣ 정산

![정산 관리 화면](screenshots/08-settlements.png)

- [ ] B/L 업로드(=거래완료) 시점에 정산 자동 생성됨
- [ ] **정산 관리** 메뉴에서 마진·정산액·실지급액 확인
- [ ] 영업담당자가 **프리랜서**면 자동으로 비율 50%, **사내직원**이면 건당 10만 원이 채워짐 (필요하면 수정 가능)
- [ ] **재무**가 검토 후 "확정" 처리 → 상태가 **"확정"**
- [ ] 지급 처리 → 상태 **"지급완료"**

#### 💰 정산 마진 공식 (영업 협상 시 참고용)

```
판매금원화        = (판매가 + Commission + 자동하역비 - TAX D/C) × 환율
정산판매금원화    = 판매금원화 - 비용9개 합계
판매마진          = 정산판매금원화 - (매입가 + 매도비)
부가세마진        = 매입가 × 9%
총마진            = (판매마진 + 부가세마진) × 0.9   ← × 0.9 = 부가세 10% 차감

정산액 (영업담당자 타입 별):
  - 프리랜서 (비율)  = 총마진 × 50%   (기본값)
  - 사내직원 (건당)  = 100,000원        (기본값)

실지급액          = 정산액 - 서류비 - 기타공제
  서류비: 프리랜서 = 50,000 / 사내직원 = 0
```

> 🔒 막힘: **"확정된 정산은 수정 불가"** 메시지
> → 재무에게 "이 정산 확정 취소해 주세요" 요청 → 수정 → 재확정

## 🆘 자주 발생하는 문제 + 빠른 해결

| 증상 | 해결 |
|---|---|
| 차량 목록에 그 차량이 안 보임 | 그 차량 담당 영업담당자가 내 휘하인지 확인. 아니면 admin에게 매핑 요청 |
| 말소신청서 RRN 칸이 비어있음 | 기본정보 탭 → 소유자 주민등록번호 수기 입력 → 서류 재생성 |
| 진행상태가 안 바뀜 | 새로고침 → 잠시 후 다시 확인 → 안 되면 개발자(진)에게 문의 |
| "수출입금 게이트 우회" 발급했는데도 막힘 | 통관용/B/L용은 별개 우회임. 두 단계 다 발급 필요 |
| 정산액이 0으로 표시 | 영업담당자의 **정산 분류**(프리랜서/사내직원) 지정됐는지 확인 (사용자 관리에서) |
| 미수금이 5~10 USD/EUR 어중간하게 남아 있음 | 십중팔구 은행 송금 수수료 누락 — 판매 탭의 "송금 수수료" 칸에 입력하면 0으로 맞춰짐 |
| 바이어 누적 수수료 카드(💸)가 안 보임 | 그 바이어 차량 중 송금 수수료 입력된 게 0건. 또는 환율 비어있음. 차량 편집 → 환율 + 송금 수수료 입력 후 저장 |
| 엑셀 컨사이니 import 시 "영업담당자 미등록" 에러 | 양식의 영업담당자 이름이 시스템 영업담당자명과 정확히 일치해야 함 |
| 차량번호 입력해도 NICE 자동조회 안 됨 | 수동 입력 가능. 안 되면 개발자에게 NICE 키 만료 여부 확인 요청 |
| 서류 다운로드가 안 됨 | 분당 30회 / 차량당 분당 10회 제한 있음. 잠시 후 재시도 |
| Ledger 영향 컬럼이 잠겨서 수정 불가 | 패널 상단 "잠금 해제" 버튼 → 사유 20자 이상 입력 → 잠금 1회 해제 (저장 후 자동 재잠금) |

## 💡 한 줄 요약

> **매입 → 말소(RRN·주소 필수) → 판매(수수료 입력) → 반입(선적) → 통관 → B/L → DHL → 거래완료 → 정산**
>
> 통관·B/L에서 막히면 **재무처리 → 수출입금 게이트 우회**.
> 입금·정산이 확정 후 수정 필요하면 **재무에 취소 요청**.
> 정산 **지급(paid) 후엔 수정 불가** — 별도 회계 처리.
