# [관리] 운영 체크리스트

차량 1대를 매입부터 정산까지 책임지고 끌고 가는 흐름. 막히는 지점마다 해제 방법 함께.

---

## 0. 처음 한 번만 — 사전 준비

- [ ] 본인 계정 권한 확인 — 역할이 **"관리"** 인지
- [ ] **사용자 관리** 메뉴 → 내 휘하 영업담당자 등록
- [ ] **포워딩사** 등록 (수출 차량에 필요)
- [ ] **바이어 / 컨사이니** 등록
  - 한두 건이면 메뉴에서 직접 추가
  - 수십~수백 건이면 엑셀 일괄 업로드 (개발자에게 양식 요청)

---

## 한 대 처리 흐름 (1~8번 순서대로)

> 📌 **워크플로우 순서**: 매입 → 말소 → 판매 → **반입(선적) → 통관 → B/L** → DHL → 거래완료 → 정산
> ※ "선적"의 의미는 **반입지 입력**. 실제 운영 순서는 차량 반입 먼저, 그 다음 통관, 마지막에 B/L 발급.

---

### 1️⃣ 차량 등록 + 매입

- [ ] **차량 관리** → "신규 등록"
- [ ] 차량번호 입력 → **NICE 자동 조회** 버튼 클릭 (24개 항목 자동 채워짐)
- [ ] **매입 탭**: 매입일·매입담당자·구입처·소유자·매입가·매도비 입력
- [ ] 비용 항목 (말소·면허·탁송·캐리·쇼링·보험·이전비·기타) 입력
- [ ] 잔금이 있으면 **잔금 추가** 버튼으로 행 추가
- [ ] 진행상태가 **"매입중"** 으로 표시되는지 확인
- [ ] 매입 잔금 다 지급 → 진행상태 **"매입완료"** 자동 전환

> 🔓 이 단계 막힘: **없음**

---

### 2️⃣ 말소

- [ ] **매입 탭** → "말소 체크" + 말소일 입력
- [ ] **⚠️ 소유자 주민등록번호 입력 확인** (말소신청서에 인쇄됨)
  - NICE 자동 조회는 RRN을 마스킹해서 주기 때문에 **차량 등록증 보고 수기 입력 필요**
  - 한 번 입력하면 암호화 저장됨
- [ ] **⚠️ 소유자 주소 입력 확인** (말소신청서·계약서에 인쇄됨, NICE 비제공)
- [ ] **서류 탭** → 말소신청서·말소계약서·위임장 자동 생성 (다운로드 → 출력)
- [ ] 말소 완료 후 말소 등록증 업로드
- [ ] 진행상태 **"말소완료"** 자동 전환

> ⚠️ 막힘:
> - 주민등록번호 비어있으면 → 말소신청서 RRN 칸이 공란으로 출력됨 → 매입 탭에서 입력 후 재생성
> - 주소 비어있으면 → 서류 주소 칸이 공란으로 출력됨 → 매입 탭에서 입력 후 재생성

---

### 3️⃣ 판매

- [ ] **판매 탭** → 판매일·통화·환율·바이어·컨사이니 선택
- [ ] 판매가·TAX D/C·Commission·운임비 입력
- [ ] 받은 입금만큼 계약금·중도금·잔금 입력
- [ ] **송금 수수료가 발생했으면 "송금 수수료" 칸에 입력** (예: USD 5,000 보냈는데 4,990 도착 + 10 수수료 → 입금 4,990 + 수수료 10)
  - 셀러(우리) 부담 처리 → 미수금 자동 차감 (바이어는 5,000 다 보낸 것으로 인정)
  - 바이어별 누적은 **바이어 편집 화면 상단** 보라색 카드에 표시됨 (영업 협상 카드로 활용)
- [ ] 진행상태 **"판매중"** → 잔금 100% 받으면 **"판매완료"** 자동 전환

> 🔒 막힘: **"확정된 입금은 수정 불가"** 메시지
> → 재무에게 "이 입금 건 확정 취소해 주세요" 요청 → 수정 → 재무가 다시 확정

> 💡 송금 수수료 누락 시: 미수금이 6 USD 같이 어중간하게 남으면 십중팔구 수수료 입력 누락. 수수료 칸에 적어서 0으로 맞추기

---

### 4️⃣ 반입 (선적 — 1차)

- [ ] **선적 탭** → **반입지 입력** + 컨테이너 번호 + VSL(선박명)
- [ ] 진행상태 **"선적중"** 자동 전환 ← 반입지가 들어가면 "선적중"

> 🔓 이 단계 막힘: **없음** (반입지 입력만)

---

### 5️⃣ 통관

- [ ] **통관 탭** → 통관 바이어·포워딩사·면장금액(USD)·도착 예정일 입력
- [ ] 운송 방식 선택 (RORO / 컨테이너), 출발 항구 입력
- [ ] **수출신고서 파일 업로드** → 진행상태 **"선적완료"** 자동 전환
- [ ] 통관 완료되면 **"통관 완료" 체크** → 진행상태 **"통관중"** 으로 표시
  - (B/L 받기 직전까지 "통관중" 상태로 유지)

> 🔒 막힘 ①: **"미수율 N% — 통관 진입 불가"** (입금 50% 미만 차량)
> → 두 가지 해제 방법:
> - (a) 입금 더 받기 → 50% 이상으로 만들기
> - (b) **재무처리 메뉴 → "수출입금 게이트 우회"** → 차량 선택 → 사유 입력 → 승인 (관리/관리자만 가능)

> 🔒 막힘 ②: **외화 차량인데 환율 미입력**
> → 판매 탭으로 가서 환율 입력 (이건 우회 불가, 환율 꼭 입력해야 함)

---

### 6️⃣ B/L (선적 — 2차)

- [ ] **선적 탭** → B/L 발급 받으면 **B/L 문서 업로드**
- [ ] 진행상태 **"거래완료"** 자동 전환 ← B/L 업로드가 거래완료 트리거

> 🔒 막힘: **"잔금 미입금 — B/L 발급 불가"** (잔금 100% 완납 필수)
> → **통관(50%)과 다름. B/L은 100% 받아야 함.**
> → 해제 방법:
> - (a) 잔금 100% 받기
> - (b) **재무처리 메뉴 → "수출입금 게이트 우회"** → 차량 선택 → 사유 명시 → 승인 (관리/관리자만 가능, 통관 우회와 **별개로 한 번 더 발급 필요**)

---

### 7️⃣ DHL

- [ ] **DHL 탭** → 수취인·발송인·중량·크기 입력
- [ ] "DHL 발송신청" 체크

> 🔓 이 단계 막힘: **없음** (정보 입력만)

---

### 8️⃣ 정산

- [ ] B/L 업로드(=거래완료) 시점에 정산 자동 생성됨
- [ ] **정산 탭** 에서 마진·정산액·실지급액 확인
- [ ] 영업담당자가 **프리랜서**면 자동으로 비율 50%, **사내직원**이면 건당 10만 원이 채워짐 (필요하면 수정 가능)
- [ ] **재무**가 검토 후 "확정" 처리 → 상태가 **"확정"**
- [ ] 지급 처리 → 상태 **"지급완료"**

> 🔒 막힘: **"확정된 정산은 수정 불가"** 메시지
> → 재무에게 "이 정산 확정 취소해 주세요" 요청 → 수정 → 재확정

---

## 📊 일상 모니터링

매일 한 번씩 확인:

- [ ] **업무 대시보드** — 내 휘하 영업담당자별 현황
- [ ] **관리자 대시보드** — 월별 매출·인원별 성과 KPI
- [ ] **채권 관리** — 미수금 위험도 (안전 / 주의 / 위험 / 심각)
- [ ] **재무 처리** — 처리 대기 건수·우회 발급 필요한 차량

---

## 🔓 락 해제 권한 정리 — 누가 풀 수 있나

| 락 종류 | **관리(나)** | 재무 | 관리자(admin) |
|---|---|---|---|
| 통관 50% 게이트 | ✅ 직접 우회 발급 | ❌ | ✅ |
| B/L 100% 게이트 | ✅ 직접 우회 발급 | ❌ | ✅ |
| 입금 확정 후 수정 | ❌ 재무에 요청 | ✅ | ❌ |
| 정산 확정 후 수정 | ❌ 재무에 요청 | ✅ | ✅ |
| 외화 환율 미입력 | (우회 불가) | (우회 불가) | (우회 불가) |
| 말소 RRN/주소 누락 | ❌ 차량 등록증 보고 수기 입력 | ❌ | ❌ |

## 🆘 자주 발생하는 문제 + 빠른 해결

| 증상 | 해결 |
|---|---|
| 차량 목록에 그 차량이 안 보임 | 그 차량 담당 영업담당자가 내 휘하인지 확인. 아니면 admin에게 매핑 요청 |
| 말소신청서 RRN 칸이 비어있음 | 매입 탭 → 소유자 주민등록번호 수기 입력 → 서류 재생성 |
| 진행상태가 안 바뀜 | 새로고침 → 잠시 후 다시 확인 → 안 되면 개발자(진)에게 문의 |
| "수출입금 게이트 우회" 발급했는데도 막힘 | 통관용/B/L용은 별개 우회임. 두 단계 다 발급 필요 |
| 정산액이 0으로 표시 | 영업담당자의 종류(프리랜서/사내직원) 지정됐는지 확인 |
| 미수금이 5~10 USD/EUR 어중간하게 남아 있음 | 십중팔구 은행 송금 수수료 누락 — 판매 탭의 "송금 수수료" 칸에 입력하면 0으로 맞춰짐 |
| 엑셀 컨사이니 import 시 "영업담당자 미등록" 에러 | 양식의 영업담당자 이름이 시스템 영업담당자명과 정확히 일치해야 함 |
| 차량번호 입력해도 NICE 자동조회 안 됨 | 수동 입력 가능. 안 되면 개발자에게 NICE 키 만료 여부 확인 요청 |
| 서류 다운로드가 안 됨 | 분당 30회 / 차량당 분당 10회 제한 있음. 잠시 후 재시도 |

---

## 💡 한 줄 요약

> **매입 → 말소(RRN·주소 필수) → 판매 → 반입(선적) → 통관 → B/L → DHL → 거래완료 → 정산**
>
> 통관·B/L에서 막히면 **재무처리 → 수출입금 게이트 우회**.
> 입금·정산이 확정 후 수정 필요하면 **재무에 취소 요청**.
