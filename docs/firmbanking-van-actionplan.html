펌뱅킹 + VAN사 도입 액션플랜

car-erp / board 에서 국내 계좌로 자동이체 (하나은행 · 국내지급 전용) · 2026-06-17

한 줄 요약 — 은행이 하나(하나은행)여도 은행이 ERP를 직접 연결해주진 않습니다. ① VAN사 가입 → ② 하나은행 펌뱅킹 계약 → ③ 테스트 → ④ 개시, 그리고 송금 안전장치(멱등·한도·화이트리스트·감사)는 car-erp 코드가 책임집니다.

전체 구조

board(영업팀장 승인) 또는 car-erp(Jin 승인) → car-erp 단일 지급 게이트웨이 → VAN사 입금이체 API → 하나은행 계좌에서 출금 → 승인된 수취계좌로 입금 → 결과 회신.
VAN 자격증명·하나은행 인증은 car-erp 한 곳에만 board는 자격증명 0 — 요청만

해야 할 일 (순서대로)

1

VAN사 선택 · 가입 사람

기간 ~1주Jin 진행
  • ERP API 연동이면 쿠콘(CODEF) 또는 웹케시가 무난 (그 외: 세틀뱅크·더즌·KS-NET 등)
  • 요청 시 "하나은행 펌뱅킹 입금이체 + 예금주조회 API + 샌드박스(테스트)" 명시
2

하나은행 펌뱅킹 계약 신청 사람

기간 1~3주Jin 진행

필요 서류 (법인 단위):

  • 펌뱅킹 계약서 · 신청서
  • 법인등기부등본
  • 법인인감증명서
  • 사업자등록증
  • 수수료 인출계좌 통장 사본
  • 대표자 실명증표(신분증)
3

전산 협의 + 샌드박스 테스트 사람 코드

기간 1~2주
  • 하나은행 전산담당 ↔ VAN ↔ car-erp 연동 테스트
  • 소액 실거래 파일럿(예: board 계약금 1건)으로 멱등·결과회신·한도 확인 후 확장
4

업무개시 전산등록 → 서비스 시작 사람

  • 테스트 완료 → 하나은행이 업무개시 등록 → 운영 이용 시작
  • 수수료: 월 1회 정산, 익월 10일(휴일이면 익영업일) 신고계좌에서 출금

car-erp 코드가 책임지는 것 (은행과 무관, 우리가 만듦)

항목내용
지급 게이트웨이 신설※ 기존 "FinanceGate"는 없음 — 신규 DisbursementService를 연동B HMAC + ApprovalRequest + AuditLog 위에 구축
멱등성거래관리번호(unique) + 상태기계(요청→처리중→성공/실패) + 이체결과조회 재확인 → 이중송금 방지
화이트리스트승인된 수취계좌에만 송금 (board=계약상대, car-erp=매입처/정산대상)
한도출처별·역할별 건당/일일 한도. 하나은행 계약 한도도 보수적으로
감사로그누가·언제·얼마·어느계좌 승인 + 실제 이체결과 한 쌍 (AuditLog)
예금주조회송금 직전 수취인 성명 확인 → 오송금 차단(권장)

⚠️ 꼭 기억할 제약

국내지급 전용. 해외 송금(수출 바이어)은 펌뱅킹으로 안 됨 → SWIFT/외환 별도 채널. 지금 자동화 대상(매입·정산·board 계약금)은 전부 국내라 OK.
멀티회사는 법인별 계약. heyman·karaba는 별도 법인·서버·.env → 각자 하나은행 계좌 + VAN 자격증명을 따로 계약·설정. 우선 heyman부터(현재 board 연동도 heyman만).
같은 서버 = 자금 보안. car-erp·board가 한 Lightsail에 있으니 그 인스턴스 보안이 곧 송금 보안. APP_KEY 보존(재생성 금지)·자격증명 분리·일 백업이 평소보다 더 중요.
