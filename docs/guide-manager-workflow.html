역할 분리(영업/수출통관/재무) 전이라, 현재는 [관리] 한 사람이 영업에게 받은 차량을 car-erp에 등록하고, 재무(입금·지급·정산)와 수출통관(반입·B/L·DHL)까지 전부 처리합니다.
→ 아래는 그 전 과정을 한 줄로 모은 통합 흐름입니다. (분리되면 가운데는 재무 가이드, 뒤는 수출통관 가이드로 넘어갑니다)

📋 통합 단계 상세

📥A. 영업 카톡 → car-erp 차량 등록관리 고유
  1. 영업이 카톡으로 보낸 차량 사진·정보를 받습니다 (차량번호·VIN·매물번호 c_no·출처(엔카/경매)·매입가·바이어 등).
  2. car-erp에 신규 차량 등록 — 기본정보·매입 정보 입력, 사진 첨부. 출처가 엔카면 엔카 매물번호도 기록.
  3. 매물번호(c_no)도 함께 기록 (첫 문의~거래완료 추적용).
※ 향후 연동 B가 되면 이 카톡·수동등록이 자동화됩니다.
💵B. 판매 등록 · 입금 처리재무 영역
  1. 판매가·통화·환율·바이어 입력 (외화는 환율 필수).
  2. 바이어 입금을 확인하면 그 금액 입력 = 자동 확정 (계약금/중도금/잔금).
  3. 입금률에 따라 다음 단계 진입이 열립니다 ↓
🚦 50% 이상 → 반입·통관 가능 / 100% 완납 → B/L 발급 가능
💰 재무 가이드 열기 →
🚢C. 반입 · 통관 · B/L · DHL수출통관 영역
  1. 수출통관 탭 — 통관 바이어/컨사이니, 포워딩사, 면장금액, 수출신고서 업로드.
  2. 선적(B/L) 탭 — 반입지·선적일·VSL 입력 → 선적 서류 생성.
  3. 100% 완납 확인 후 B/L 발급 (번호·컨테이너·B/L 문서) → 거래완료.
  4. DHL 탭 — 수취인 정보·발송신청.
⚠️ B/L은 100% 완납 필수. 미완납은 본인(관리) 권한으로 승인 우회 가능하나 신중히. (수출 채널만 해당)
🚢 수출통관 가이드 열기 →
📊D. 정산 확정 · 지급 · 2차 정산재무 영역
  1. 거래완료 시 정산 자동 생성(pending) → 금액 검토 후 확정(confirmed).
  2. 지급(paid) 처리 — 현재는 [관리]가 직접 (분리되면 재무는 확정까지, 지급은 관리 승인).
  3. 지급 후 한 달 내 2차 정산 — 실측 비용 보정·환차 정정 → 2차 완료(closed).
🔐 역할 분리 시 — 재무는 confirmed까지, paid(지급)는 관리 승인. 지금은 한 사람이라 통합 처리하되, 분리 대비해 흐름은 동일하게 유지.
💰 재무 가이드 열기 →
🛡️E. 관리 고유 권한 (분리돼도 관리 몫)관리 전용
  1. 정산 지급(paid) 승인 — 재무 확정 건을 지급 처리.
  2. 게이트 우회 승인 — 미완납 B/L 발급 등 예외 승인.
  3. 마감 해제·삭제 등 민감 액션 (감사로그 기록됨).

⚠️ 통합 운영 시 핵심 주의

  1. 등록 시 VIN·매물번호(c_no) 누락 금지 — 추적·중복 방지의 핵심.
  2. 외화차 환율 항상 입력 — 미수금 오판 방지.
  3. 입금률 게이트(50%/100%) 확인 후 통관·B/L 진행.
  4. 한 사람이 다 하더라도 확정→지급 순서·스냅샷 잠금 흐름은 그대로 (분리 대비).
