## 2. 명칭 확정 (최종)

대시보드는 3개 URL로 유지. `/erp/work-dashboard` 별도 신설 없이 **업무 대시보드 안에 토글로 흡수**한다.

| 명칭 | URL | 대상 | 성격 |
|---|---|---|---|
| **업무 대시보드** | `/erp/dashboard` | admin/super | `[담당자별]` ↔ `[역할별]` 토글로 두 뷰 전환 |
| **일반사용자 대시보드** | `/erp/dashboard` | 일반 user | role별 오늘의 할일 |
| **관리자 대시보드** | `/admin/dashboard` | admin/super | 기간별 결과/성과/차트 |

> `/erp/dashboard`는 로그인한 사용자의 permission에 따라 화면이 다름.
> - admin → **업무 대시보드** (담당자 드롭다운 + 담당자별/역할별 토글)
> - 일반 user → **일반사용자 대시보드** (role별 할일)
> - 두 화면이 같은 URL을 공유하므로 코드 내에서 `isAdmin()` 분기로 처리.

### 업무 대시보드 토글 구조

```
[담당자별 보기]  |  [역할별 보기]
```

- **담당자별 보기**: 담당자 드롭다운 선택 → 해당 담당자의 처리 필요 항목 + 진행중 차량 (현재 구조)
- **역할별 보기**: 탭 구조 → 영업 업무 / 수출통관 업무 / 재무 업무 / 채권 업무 / 문서 업무

## 3. 차량 흐름도 (확정)

`Vehicle::progress_status` 계산 기준과 일치하는 흐름:

```
매입중
  ↓ 매입가 입력 + 매입 미지급 0
매입완료
  ↓ 말소 체크 + 말소서류 업로드
말소완료
  ↓ 판매가 입력
판매중
  ↓ 판매 미입금 0
판매완료
  ↓ 통관 바이어 + 선적일 입력
수출통관중
  ↓ 수출신고서 업로드
수출통관완료
  ↓ 반입지 입력
선적중
  ↓ B/L 문서 업로드
선적완료
  ↓ DHL 발송신청 체크
거래완료

※ 큐 17 — 폐기 단계 제거. 운영상 차량 폐기 없음 (10단계로 단순화).
```

### role별 책임 구간

| 단계 | 주 책임 | 대시보드 action |
|---|---|---|
| 매입중 / 매입완료 | 영업 | 매입가 입력, 미지급 확인 |
| 말소완료 | 영업 | 판매 등록 필요 |
| 판매중 / 판매완료 | 영업 + 재무 | 미입금 확인, 바이어 누락 |
| 수출통관중 / 완료 | 수출통관 | 수출신고서 업로드 |
| 선적중 / 선적완료 | 수출통관 | B/L 업로드, DHL 발송 |
| 거래완료 | 재무 | 정산 생성/확정/지급 |

## 4. 파이프라인 스트립 (2종)

### 4-1. 대시보드용 카운트 스트립

모든 대시보드 상단에 공통으로 들어가며, 범위만 다르다.

```
[매입중] → [매입완료] → [말소완료] → [판매중] → [판매완료]
  (5)        (5)          (4)          (8)          (5)
→ [수출통관중] → [수출통관완료] → [선적중] → [선적완료] → [거래완료]
     (5)            (5)            (5)          (4)          (4)
```

- 색상: 뱃지 체계 그대로 (파랑/보라/주황/초록/회색)
- 클릭 → `progressFilter`로 차량 목록 이동
- 대시보드별 범위:
  - 일반사용자(영업): 본인 salesman 차량만
  - 일반사용자(수출통관/재무): 전체 차량
  - 업무 대시보드: 선택된 담당자 기준
  - 관리자 대시보드: 전체 차량

### 4-2. 차량 상세용 상태 스트립 (신규)

차량 편집 패널 상단에 해당 차량 1대의 단계별 완료/대기/문제 상태 표시.

```
매입완료 ✓ → 말소완료 ✓ → 판매완료 ✓ → 입금경고 ! → 통관대기 - → 선적대기 - → DHL대기 -
```

- ✓ 완료 / ! 경고(미입금 등) / - 대기
- 각 노드 클릭 → 해당 탭으로 이동

| 노드 | 이동 탭 |
|---|---|
| 매입 | 매입 탭 |
| 말소 | 매입/서류 탭 |
| 판매/입금 | 판매 탭 또는 채권관리 |
| 통관 | 수출통관 탭 |
| 선적/B/L | 선적 탭 |
| DHL | DHL 탭 |

## 5. 일반사용자 대시보드 — role별 분기

현재 `/erp/dashboard`는 role 구분 없이 동일한 actionCounts를 보여줌. 아래 기준으로 분기 필요.

### 5-1. 영업 role (현재 구조 유지 + 보강)

**KPI 카드:** 진행중 / 이달 매입 / 이달 판매 / 이달 거래완료

**오늘의 할일:**

| action | 조건 |
|---|---|
| 매입 미지급 | purchase_unpaid_amount > 0 |
| 말소 처리 필요 | 매입완료인데 말소서류 없음 |
| 판매 등록 필요 | 말소완료인데 sale_price = 0 |
| 판매 미입금 | sale_unpaid_amount > 0 |
| 바이어/컨사이니 누락 | 판매중인데 buyer 미연결 |

**파이프라인:** 본인 salesman 차량만

---

### 5-2. 수출통관 role (신규 분기 필요)

**KPI 카드:** 통관 신청 대기 / 수출신고서 업로드 대기 / 선적 처리 대기 / DHL 발송 대기

**오늘의 할일:**

| action | 조건 |
|---|---|
| 수출통관 신청 필요 | 판매완료인데 export_declaration_document 없음 |
| 통관 정보 누락 | export_buyer/shipping_date 미입력 |
| 포워딩사 미지정 | 통관 단계인데 forwarding_company 없음 |
| 수출신고서 업로드 필요 | 수출통관중인데 수출신고서 없음 |
| 선적 처리 필요 | 수출통관완료인데 bl_loading_location 없음 |
| B/L 업로드 필요 | 선적중인데 bl_document 없음 |
| DHL 발송 필요 | 선적완료인데 dhl_request = false |

**파이프라인:** 전체 차량 (통관 이후 단계 강조)

---

### 5-3. 재무 role (신규 분기 필요)

**KPI 카드:** 매입 미지급 총액 / 판매 미입금 총액 / 정산 대기 건수 / 환율 미입력 외화 차량

**오늘의 할일:**

| action | 조건 |
|---|---|
| 매입 미지급 | purchase_unpaid_amount > 0 |
| 판매 미입금 | sale_unpaid_amount > 0 |
| 환율 입력 필요 | currency != KRW + sale_price > 0 + exchange_rate 없음 |
| 정산 생성 필요 | 거래완료인데 settlement 없음 |
| 정산 확정 필요 | settlement_status = pending |
| 정산 지급 필요 | settlement_status = confirmed |
| 채권 위험 | receivable_risk = danger 또는 critical |

**파이프라인:** 전체 차량 (입금/정산 관련 단계 강조)

## 10. 구현 우선순위 (수정본)

### 1단계 — 일반사용자 대시보드 role 분기
`/erp/dashboard`에서 `role` 확인 후 영업/수출통관/재무 KPI + 할일 목록 분기

### 2단계 — 파이프라인 카운트 스트립
대시보드용 공통 컴포넌트. 업무 대시보드 + 일반사용자 대시보드에 먼저 적용.

### 3단계 — 차량관리 담당자 필터 + 채권관리 검색란 축소
소소한 UX 수정.

### 4단계 — 관리자 대시보드 차트 보강
연간 월별 차트, 담당자별 성과, role 탭.

### 5단계 — 운영 대시보드 신설 (`/erp/work-dashboard`)
role별 업무 큐 탭 구조.

### 6단계 — 차량 상세 흐름도 스트립
차량 편집 패널 상단 1대용 상태 스트립.

### 7단계 — 권한 세분화
문서 다운로드 제한, 민감정보 마스킹.

### 8단계 — NICE API 연동
위 모든 것이 완성된 뒤.
